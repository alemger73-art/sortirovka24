КРИТИЧЕСКАЯ ЗАДАЧА: нет доступа в админ-панель.

Нужно настроить и гарантировать рабочий вход в админку в production.

==================================================

1. АДРЕС АДМИНКИ

Проверить и настроить:

- админ-панель должна открываться по адресу:
  /admin

- полный URL:
  https://<домен>/admin

Если маршрут не настроен — создать.

==================================================

2. СОЗДАТЬ ПОЛЬЗОВАТЕЛЯ

Создать администратора в базе данных:

Логин: admin

Пароли (разрешить вход любым из):

- Admin
- Adminger123@
- Adminger123 (если спецсимволы ломают авторизацию)

ВАЖНО:

- пароль должен корректно хешироваться
- проверить соответствие bcrypt/argon
- убедиться, что авторизация реально проходит

==================================================

3. АВТОРИЗАЦИЯ

Проверить:

- login endpoint работает
- JWT токен создаётся
- токен сохраняется
- после входа происходит redirect в админку
- нет бесконечного logout

==================================================

4. ПРОВЕРИТЬ PRODUCTION

Обязательно протестировать:

- вход с внешнего сайта (не App Viewer)
- вход с телефона
- вход после перезагрузки страницы

==================================================

5. COOKIE / TOKEN

Проверить:

- secure cookies
- sameSite настройки
- нет блокировки авторизации в production

==================================================

6. FALLBACK (ОБЯЗАТЕЛЬНО)

Если что-то не работает:

- сделать временный bypass:
  - при переходе на /admin → автоматический вход как admin

(для теста, потом можно убрать)

==================================================

7. ВЫВОД

После выполнения:

- указать точный URL админки
- подтвердить, что вход работает
- подтвердить, что пользователь создан

==================================================
